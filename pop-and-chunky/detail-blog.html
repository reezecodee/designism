Cara Setup CI/CD Pipeline Jenkins untuk Project Laravel

Portfolio
| 14 JAN 2026

Deployment manual itu melelahkan. Bayangkan setiap kali ada update kecil di backend Laravel, kita harus login SSH, `git pull`, jalankan migrasi, dan restart service. Di artikel ini, saya akan berbagi pengalaman setup Jenkins untuk mengotomatisasi proses tersebut.

1. Persiapan Server

Pastikan server VPS (saya pakai Ubuntu 22.04) sudah terinstall Java, karena Jenkins berjalan di atas Java Runtime Environment (JRE). 

sudo apt update
sudo apt install fontconfig openjdk-17-jre
java -version

2. Install Jenkins

Selanjutnya, kita tambahkan repository Jenkins ke sistem dan lakukan instalasi. Jangan lupa untuk membuka port 8080 di firewall jika belum dibuka. 

sudo wget -O /usr/share/keyrings/jenkins-keyring.asc \
  https://pkg.jenkins.io/debian-stable/jenkins.io-2023.key
  
sudo apt-get install jenkins
sudo systemctl start jenkins

3. Konfigurasi Pipeline

Di dashboard Jenkins, buat "New Item" > "Pipeline". Di bagian konfigurasi script, kita bisa menggunakan Jenkinsfile. Berikut adalah contoh script sederhana untuk Laravel:

ðŸ’¡ Catatan: Pastikan Anda sudah mengatur SSH Credentials di Jenkins agar bisa connect ke server production. 
pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                sh 'composer install --no-dev'
                sh 'cp .env.example .env'
                sh 'php artisan key:generate'
            }
        }
        stage('Test') {
            steps {
                sh './vendor/bin/phpunit'
            }
        }
    }
}

Kesimpulan

Dengan setup ini, setiap kali saya push code ke GitHub, Jenkins akan otomatis menjalankan test. Jika lolos, code siap dideploy. Ini menghemat waktu saya sekitar 15-20 menit per deployment.

Next, saya akan bahas cara setup Webhook agar trigger-nya otomatis!
